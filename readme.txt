Название задачи: Numberlink

Описание
Программа построена на расширяемой архитектуре и допускает добавление новых типов геометрии.
В текущей версии реализована поддержка шестиугольного поля (hex).

Поддерживаемые возможности
- чтение головоломки из текстового файла
- проверка корректности формата входных данных
- проверка логической корректности головоломки
- построение соседей для шестиугольной сетки
- поиск одного или нескольких решений
- вывод найденных решений в консоль
- справка по ключам -h и --help
- режим отладки через ключ --debug

Формат входного файла
В первой строке указывается тип геометрии:
hex

Далее идут строки поля.
Клетки в строке разделяются пробелами.
Пустая клетка обозначается символом .
Конечные точки обозначаются одинаковыми буквами.

Пример входного файла:
hex
A . .
. B .
A . B

Пример запуска
1. Запуск с файлом по умолчанию:
python -m numberlink.main

2. Запуск с указанием файла:
python -m numberlink.main puzzle.txt

3. Показ справки:
python -m numberlink.main --help

4. Запуск в отладочном режиме:
python -m numberlink.main puzzle.txt --debug

Пример вывода
Найдено 1 решение:
Решение 1:
A A B
A B B
A . B

Структура программы
numberlink/
    main.py                - точка входа, CLI, обработка ошибок
    exceptions.py          - пользовательские исключения
    render.py              - вывод решения в консоль

    domain/
        position.py        - класс позиции клетки
        puzzle.py          - модель головоломки
        state.py           - состояние поиска (если используется)

    io/
        loader.py          - загрузка текста из файла
        parser.py          - разбор входного текста в объект Puzzle

    geometry/
        factory.py         - создание объекта геометрии
        hex.py             - логика соседей для hex-поля

    solver/
        base.py            - базовый интерфейс решателя
        backtracking.py    - backtracking/DFS решатель

    validation/
        validator.py       - логическая проверка головоломки

Основные модули
- parser.py преобразует текст файла во внутреннюю модель Puzzle
- validator.py проверяет, что поле корректно и каждая буква встречается ровно 2 раза
- hex.py определяет соседние клетки для шестиугольной сетки
- backtracking.py ищет решения с помощью рекурсивного перебора с откатом
- render.py отвечает за красивый вывод решения

Обработка ошибок
Программа обрабатывает типичные ошибки пользователя:
- файл не найден
- входной файл пуст
- строки поля имеют разную длину
- встречен недопустимый символ
- указана неподдерживаемая геометрия
- у буквы неверное количество конечных точек

В обычном режиме программа не выводит Traceback пользователю.
Подробный Traceback допускается только при запуске с ключом --debug.

Особенности реализации
- используется декомпозиция по модулям
- логика решения отделена от пользовательского интерфейса
- поиск решений реализован через backtracking
- программа работает как консольное приложение

Ограничения текущей версии
- в текущей версии реализована только геометрия hex
- для добавления новых геометрий нужно реализовать соответствующий модуль геометрии и подключить его через фабрику
- вывод решения текстовый, без графического интерфейса
- сложные головоломки могут решаться долго из-за полного перебора
